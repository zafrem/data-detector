# Data Detector 아키텍처

이 문서는 텍스트 기반 다이어그램을 사용하여 data-detector 시스템의 아키텍처를 설명합니다.

## 구성 요소 세부 사항

### 1. 인터페이스 계층

```
┌─────────────────────────────────────────────────────┐
│                  CLI (Click)                        │
├─────────────────────────────────────────────────────┤
│ 명령:                                                │
│  • find        - 텍스트에서 PII 찾기                    │
│  • validate    - 패턴에 대해 텍스트 검증                 │
│  • redact      - 텍스트/파일에서 PII 수정                │
│  • list-patterns - 사용 가능한 패턴 나열                 │
│  • serve       - HTTP/gRPC 서버 시작                  │
│  • version     - 버전 표시                            │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│              Library API (Python)                   │
├─────────────────────────────────────────────────────┤
│ from datadetector import Engine, load_registry      │
│                                                     │
│ registry = load_registry(paths=["patterns/"])       │
│ engine = Engine(registry)                           │
│                                                     │
│ # 작업                                               │
│ engine.find(text, namespaces=[...])                 │
│ engine.validate(text, pattern_id)                   │
│ engine.redact(text, namespaces=[...])               │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│          Server (HTTP/gRPC)                         │
├─────────────────────────────────────────────────────┤
│ 엔드포인트:                                            │
│  POST /find      - PII 찾기                          │
│  POST /validate  - 패턴 검증                          │
│  POST /redact    - PII 수정                          │
│  GET  /patterns  - 패턴 나열                          │
│  GET  /health    - 상태 확인                          │
│  GET  /metrics   - Prometheus 메트릭                 │
│                                                     │
│ 기능:                                                │
│  • TLS 지원                                          │
│  • 속도 제한                                          │
│  • 핫 리로드                                          │
└─────────────────────────────────────────────────────┘
```

### 2. 핵심 엔진 계층

```
┌──────────────────────────────────────────────────────────┐
│                       Engine                             │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  find(text, namespaces, include_matched_text)            │
│  ┌────────────────────────────────────────────┐          │
│  │ 1. 레지스트리에서 패턴 가져오기                   │          │
│  │ 2. 텍스트에 각 패턴 적용                        │          │
│  │ 3. 있으면 검증 함수 실행                        │          │
│  │ 4. 메타데이터와 함께 일치 수집                    │          │
│  │ 5. FindResult 반환                          │          │
│  └────────────────────────────────────────────┘          │
│                                                          │
│  validate(text, pattern_id)                              │
│  ┌────────────────────────────────────────────┐          │
│  │ 1. 레지스트리에서 단일 패턴 가져오기               │          │
│  │ 2. 텍스트가 패턴과 일치하는지 확인                 │          │
│  │ 3. 있으면 검증 함수 실행                        │          │
│  │ 4. ValidationResult (bool) 반환             │          │
│  └────────────────────────────────────────────┘          │
│                                                          │
│  redact(text, namespaces, strategy)                      │
│  ┌────────────────────────────────────────────┐          │
│  │ 1. 텍스트에서 모든 일치 찾기                     │          │
│  │ 2. 수정 전략 적용:                            │          │
│  │    - mask: 마스크 값으로 교체                   │          │
│  │    - hash: 해시로 교체                        │          │
│  │    - tokenize: 토큰으로 교체                   │          │
│  │ 3. 수정된 텍스트 구축                           │          │
│  │ 4. RedactionResult 반환                     │          │
│  └────────────────────────────────────────────┘          │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│                  PatternRegistry                         │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  데이터 구조:                                               │
│  ┌──────────────────────────────────────────────┐        │
│  │ _patterns: Dict[str, Pattern]                │        │
│  │   키: "namespace/pattern_id"                  │        │
│  │   값: Pattern 객체                             │        │
│  │                                              │        │
│  │ _by_namespace: Dict[str, List[Pattern]]      │        │
│  │   키: "namespace"                             │        │
│  │   값: 패턴 목록                                 │        │
│  │                                              │        │
│  │ _verification_funcs: Dict[str, Callable]     │        │
│  │   키: "function_name"                         │        │
│  │   값: 검증 함수                                 │        │
│  └──────────────────────────────────────────────┘        │
│                                                          │
│  메서드:                                                   │
│  • register_pattern(pattern)                             │
│  • get_pattern(pattern_id)                               │
│  • get_patterns_by_namespace(namespace)                  │
│  • register_verification(name, func)                     │
│  • list_namespaces()                                     │
└──────────────────────────────────────────────────────────┘
```

### 6. 패턴 파일 구조

```yaml
# 패턴 파일 (YAML)
┌─────────────────────────────────────────────┐
│ namespace: "kr"                             │  # 필수
│ description: "Korean patterns"              │  # 필수
│                                             │
│ patterns:                                   │  # 필수 (목록)
│   - id: "mobile_01"                         │  # 필수
│     location: "kr"                          │  # 필수
│     category: "phone"                       │  # 필수 (enum)
│     description: "Korean mobile"            │  # 선택
│     pattern: "01[0-9]-\\d{4}-\\d{4}"        │  # 필수 (regex)
│     mask: "***-****-****"                   │  # 선택
│     flags: ["IGNORECASE"]                   │  # 선택
│     verification: "luhn"                    │  # 선택
│                                             │
│     examples:                               │  # 선택
│       match:                                │
│         - "010-1234-5678"                   │
│       nomatch:                              │
│         - "02-1234-5678"                    │
│                                             │
│     policy:                                 │  # 필수
│       store_raw: false                      │  # 필수
│       action_on_match: "redact"             │  # 필수 (enum)
│       severity: "high"                      │  # 필수 (enum)
│                                             │
│   - id: "rrn_01"                            │
│     # ... 다음 패턴                           │
└─────────────────────────────────────────────┘
```

## 핵심 설계 원칙

### 1. 관심사 분리
- **인터페이스 계층**: 사용자 상호작용 (CLI/API/Server)
- **핵심 엔진**: 비즈니스 로직 (find/validate/redact)
- **레지스트리**: 패턴 관리 및 캐싱
- **저장소**: YAML 파일 영속성
- **유틸리티**: 재사용 가능한 YAML 작업

### 2. 확장성
- 검증 함수를 위한 플러그인 아키텍처
- 네임스페이스 기반 패턴 구성
- YAML 유틸리티를 통한 사용자 정의 패턴 파일 지원
- 다중 인터페이스 옵션 (CLI/Library/Server)

### 3. 성능
- 레지스트리의 컴파일된 정규식 캐싱
- 네임스페이스 기반 필터링
- 효율적인 패턴 매칭 (1KB에 대해 p95 < 10ms)
- 다운타임 없는 핫 리로드

### 4. 보안 및 개인정보
- 로그에 원시 PII 없음 (해시만)
- 정책 기반 데이터 처리 (store_raw 플래그)
- 서버용 TLS 지원
- 설정 가능한 심각도 수준

### 5. 개발자 경험
- 간단한 API (3개의 주요 메서드가 있는 Engine)
- 패턴 관리를 위한 포괄적인 YAML 유틸리티
- dataclass를 사용한 강력한 타이핑
- 광범위한 문서 및 예제
- 94% 테스트 커버리지
